# YouTube動画脚本 TTS変換ガイドライン

このガイドラインは、Claude Codeが脚本をYouTube動画向けの自然で親しみやすい発話テキストに変換する際の指針です。ElevenLabs TTSの機能を最大限に活用し、視聴者に響くナレーションを実現します。

---

## 1. 基本原則

### 1.1 話し言葉として書く

脚本は「読む」ためではなく「聴く」ために最適化します。

**避けるべき書き方:**
```
本研究では、FourCastNetを用いた気象予測手法について検討を行った。
```

**推奨する書き方:**
```
今回は、FourCastNetっていうAIを使って、天気予報をやってみました。
```

### 1.2 一文は短く

- 一文は20〜30文字程度を目安に
- 複雑な文は複数の短い文に分割
- 主語と述語を近づける

---

## 2. 間（ポーズ）のコントロール

### 2.1 句読点による自然な間

| 記号 | 効果 | 使用例 |
|------|------|--------|
| `、` | 短い息継ぎ | 「今日は、天気の話をします」 |
| `。` | 文の終わり、標準的な間 | 「これが結果です。では次に進みましょう」 |
| `…` | ためらい、余韻、思考中 | 「それは…ちょっと難しいですね」 |
| `―` または `—` | 短い間、補足の挿入 | 「これは―実は重要なポイントで―見逃せません」 |

### 2.2 SSMLブレイクタグ（v2モデル用）

より精密な間のコントロールが必要な場合：

```
「ここで重要なのは」<break time="1.0s" />「データの質です」
```

**注意:**
- Eleven v3ではSSMLブレイクタグは非対応
- 1回の生成で使いすぎると不安定になる可能性あり

### 2.3 文構造による間の演出

強調したいポイントの前後に改行を入れることで、自然な間を作れます：

```
そして、ここからが本題です。

AIが予測した結果を見てみましょう。
```

## 3. 感情・トーンの表現

### 3.1 ナラティブスタイル

感情は文脈や説明で伝えます：

```
「これ、すごくないですか？」と、思わず声が大きくなってしまうような結果が出ました。
```

### 3.2 Audio Tags（Eleven v3用）

v3モデルでは角括弧 `[]` でタグを挿入できます：

#### 感情タグ
| タグ | 効果 |
|------|------|
| `[excited]` | 興奮、ワクワク |
| `[curious]` | 好奇心、疑問 |
| `[surprised]` | 驚き |
| `[sad]` | 悲しみ |
| `[sarcastic]` | 皮肉っぽく |

#### 発話スタイルタグ
| タグ | 効果 |
|------|------|
| `[whispers]` | ささやき |
| `[laughs]` | 笑い |
| `[sighs]` | ため息 |
| `[clears throat]` | 咳払い |

#### 使用例
```
[excited] みなさん、これ見てください！

[curious] でも、なぜこうなるんでしょうか…

[laughs] 実は私も最初は信じられませんでした。
```

### 3.3 大文字による強調

重要な単語は大文字化で強調できます：

```
これはVERY importantなポイントです。
```

日本語の場合は、カタカナや記号で強調：

```
これは「とても」重要なポイントです！
```

## 4. YouTube動画向けの親しみやすさ

### 4.1 視聴者への呼びかけ

```
みなさん、こんにちは！
今回の動画では…

ここまで見てくれた方、ありがとうございます！
```

### 4.2 口語表現の活用

| 硬い表現 | 親しみやすい表現 |
|----------|------------------|
| ～である | ～なんですね |
| ～と考えられる | ～かもしれません |
| ～を実施した | ～をやってみました |
| 興味深い結果 | 面白い結果 |
| 検討する | 考えてみる |

### 4.3 聞き手を巻き込む

```
ここで質問です。みなさんはどう思いますか？

…正解は、こちらでした！
```

### 4.4 リアクションの挿入

```
結果を見てみると…おお、これはすごい！

予想と違いましたね。ちょっと意外でした。
```

## 5. 専門用語・数値の扱い

### 5.1 専門用語は補足説明を

```
FourCastNet、つまりAI気象予測モデルを使います。

解像度は0.25度、これはだいたい25キロメートル四方です。
```

### 5.2 数値の読み上げ

TTSが正しく読めるように変換します：

| 元の表記 | TTS用表記 |
|----------|-----------|
| `0.25°` | `0.25度` または `零点二五度` |
| `10km` | `10キロメートル` または `じゅっキロ` |
| `2024/01/15` | `2024年1月15日` |
| `50%` | `50パーセント` |
| `$100` | `100ドル` |

### 5.3 略語・頭字語

```
<!-- 避ける -->
GCMを使用します。

<!-- 推奨 -->
GCM、つまり全球気候モデルを使用します。
```

## 6. 文章構成のテンプレート

### 6.1 導入部

```
[friendly tone] みなさん、こんにちは！

今回は[トピック]について解説していきます。

この動画を見ると、[得られるメリット]がわかります。

では、さっそく始めていきましょう！
```

### 6.2 セクション移行

```
はい、ここまでが[前のセクション]の話でした。

[短い間]

続いて、[次のセクション]を見ていきましょう。
```

### 6.3 締めくくり

```
今回のまとめです。

ポイントは3つありました。
1つ目は…
2つ目は…
そして3つ目は…

この動画が参考になったら、高評価とチャンネル登録をお願いします！

それでは、また次の動画でお会いしましょう。
```

## 7. 発音の制御

### 7.1 発音辞書の活用

頻出する専門用語は発音辞書で統一：

```xml
<lexeme>
  <grapheme>FourCastNet</grapheme>
  <alias>フォーキャストネット</alias>
</lexeme>
```

### 7.2 読み仮名の直接指定

発音辞書を使わない場合は、テキスト内で指定：

```
<!-- 方法1: 括弧で補足 -->
ERA5（イーラファイブ）のデータを使います。

<!-- 方法2: 読み仮名に置換 -->
イーラファイブのデータを使います。
```

## 8. 品質チェックリスト

脚本完成時に以下を確認：

- [ ] 一文が長すぎないか（30文字以内が目安）
- [ ] 専門用語に補足説明があるか
- [ ] 数値・単位は読み上げ形式になっているか
- [ ] 適切な間（ポーズ）が入っているか
- [ ] 視聴者への呼びかけがあるか
- [ ] 口語調で書かれているか
- [ ] Audio Tagsは適切に使用されているか（v3の場合）
- [ ] 導入・締めくくりは親しみやすいか

## 9. ElevenLabs設定の推奨値

### 9.1 Stability（安定性）

| 設定 | 用途 |
|------|------|
| Creative（低め） | 感情豊かな表現、Audio Tags活用時 |
| Natural（中央） | バランスの取れた自然な発話 |
| Robust（高め） | 安定した読み上げ、ニュース調 |

### 9.2 Speed（速度）

- デフォルト: `1.0`
- 説明パート: `0.9`〜`1.0`（ゆっくりめ）
- 盛り上がり部分: `1.0`〜`1.1`（やや速め）

## 10. 避けるべきパターン

### 10.1 TTSが苦手な表現

```
<!-- 避ける -->
（笑）、www、ww
→ 発話されないか、文字通り読まれる

<!-- 代替 -->
[laughs] または ふふっ
```

### 10.2 曖昧な記号

```
<!-- 避ける -->
〜、♪、★、※

<!-- 代替 -->
明示的な言葉に置き換える
```

### 10.3 視覚的な表現

```
<!-- 避ける -->
↑この図を見てください

<!-- 代替 -->
画面に表示されている図を見てください
```
